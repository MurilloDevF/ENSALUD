# Garantías y Soporte – ENSALUD Presupuestal

Versión: 1.0  
Fecha: 2026-02-08  
Alcance: Aplicativo "ENSALUD Presupuestal".

## 1. Objetivo
Definir condiciones de soporte inicial y garantía de funcionalidad del código.

## 2. Soporte inicial (1 mes)
- Onboarding y corrección de defectos.  
- SLA: P1 ≤4h, P2 ≤8h, P3 ≤16h, P4 ≤48h.

## 3. Garantía (6 meses)
- Cubre defectos de funcionalidad entregada.  
- No cubre malas prácticas del usuario (edición manual de CSV, formatos no soportados, ejecución en ambientes no soportados, etc.).

## 4. Procedimiento de incidentes
Reportar vía GitHub Issues con pasos, evidencias y archivos.

## 5. Límites
Responsabilidad limitada; exclusiones por terceros y hardware/OS.
